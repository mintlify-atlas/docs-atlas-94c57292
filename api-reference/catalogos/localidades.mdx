## Overview

Localidades (localities) are geographic entities associated with municipalities. This catalog provides reference data for localities within the Yucatan region.

## Get All Localidades

<api-endpoint>
  <api-endpoint-request>
    GET /localidad
  </api-endpoint-request>
</api-endpoint>

Retrieves a list of all localities in the system.

### Authentication

This endpoint requires authentication using a Bearer token.

### Response

<ResponseField name="id" type="number" required>
  Unique identifier for the locality
</ResponseField>

<ResponseField name="nombre" type="string" required>
  Name of the locality (max 100 characters)
</ResponseField>

<ResponseField name="clave" type="string">
  Optional code or key for the locality
</ResponseField>

<ResponseField name="activo" type="boolean" required>
  Status flag indicating if the locality is active (default: true)
</ResponseField>

<ResponseField name="municipioId" type="number" required>
  ID of the municipality this locality belongs to
</ResponseField>

<ResponseField name="municipio" type="object">
  Related municipality entity object
</ResponseField>

### Example Request

```bash
curl --request GET \
  --url http://localhost:3000/localidad \
  --header 'Authorization: Bearer YOUR_TOKEN'
```

### Example Response

```json
[
  {
    "id": 1,
    "nombre": "Merida Centro",
    "clave": "MER-01",
    "activo": true,
    "municipioId": 1,
    "municipio": {
      "id": 1,
      "nombre": "Merida"
    }
  },
  {
    "id": 2,
    "nombre": "Valladolid Norte",
    "clave": "VAL-01",
    "activo": true,
    "municipioId": 2,
    "municipio": {
      "id": 2,
      "nombre": "Valladolid"
    }
  }
]
```

  <api-endpoint-request>

Retrieves a single locality by its unique identifier.

This endpoint requires authentication using a Bearer token.

### Path Parameters

<ParamField path="id" type="number" required>

</ParamField>

### Response

Returns a single locality object with the following fields:

<ResponseField name="id" type="number" required>
  Unique identifier for the locality
</ResponseField>

<ResponseField name="nombre" type="string" required>
  Name of the locality
</ResponseField>

<ResponseField name="clave" type="string">
  Optional code or key for the locality
</ResponseField>

<ResponseField name="activo" type="boolean" required>
  Status flag indicating if the locality is active
</ResponseField>

<ResponseField name="municipioId" type="number" required>
  ID of the municipality this locality belongs to
</ResponseField>

<ResponseField name="municipio" type="object">
  Related municipality entity object
</ResponseField>

### Example Request

```bash
curl --request GET \
  --url http://localhost:3000/localidad/1 \
  --header 'Authorization: Bearer YOUR_TOKEN'
```

```json
{
  "id": 1,

  "municipioId": 1,
```

## Create Localidad

<api-endpoint>
  <api-endpoint-request>
    POST /localidad
  </api-endpoint-request>
</api-endpoint>

Creates a new locality in the system.

### Authentication

This endpoint requires authentication using a Bearer token.

### Request Body

<ParamField body="nombre" type="string" required>
  Name of the locality (max 100 characters)
</ParamField>

<ParamField body="clave" type="string">
  Optional code or key for the locality
</ParamField>

<ParamField body="municipioId" type="number" required>
  ID of the municipality this locality belongs to
</ParamField>

<ParamField body="activo" type="boolean">
  Status flag (defaults to true if not provided)
</ParamField>

### Example Request

```bash
curl --request POST \
  --url http://localhost:3000/localidad \
  --header 'Authorization: Bearer YOUR_TOKEN' \
  --header 'Content-Type: application/json' \
  --data '{
    "nombre": "Tizimin Sur",
    "clave": "TIZ-02",
    "municipioId": 3,
    "activo": true
  }'
```

### Example Response

```json
{
  "id": 3,
  "nombre": "Tizimin Sur",
  "clave": "TIZ-02",
  "activo": true,
  "municipioId": 3
}
```

## Update Localidad

<api-endpoint>
  <api-endpoint-request>
    PATCH /localidad/{id}
  </api-endpoint-request>
</api-endpoint>

Updates an existing locality.

### Authentication

This endpoint requires authentication using a Bearer token.

### Path Parameters

<ParamField path="id" type="number" required>
  The unique identifier of the locality to update
</ParamField>

### Request Body

All fields are optional. Only include fields you want to update.

<ParamField body="nombre" type="string">
  Name of the locality
</ParamField>

<ParamField body="clave" type="string">
  Code or key for the locality
</ParamField>

<ParamField body="municipioId" type="number">
  ID of the municipality
</ParamField>

<ParamField body="activo" type="boolean">
  Status flag
</ParamField>

### Example Request

```bash
curl --request PATCH \
  --url http://localhost:3000/localidad/3 \
  --header 'Authorization: Bearer YOUR_TOKEN' \
  --header 'Content-Type: application/json' \
  --data '{
    "activo": false
  }'
```
